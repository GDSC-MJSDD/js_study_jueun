10장 객체 리터럴 자바스크립트: 객체 기반의 언어 데이터 타입 - 원시 타입: 단 하나의 값, 불변 - 객체 타입: 자료구조, 가변 객체: 0개 이상의 프로퍼티(키:값;)로 구성. 함수도 프로퍼티가 될 수 있다. *프로퍼티인 함수: 메서드(일반함수와 구분하기 위한 이름) 자바스트립트는 프로토타입 기반 객체지향 언어 (cf. java: 클래스 기반 객체지향언어) js의 객체 생성 방법: 객체 리터럴, Object생성자 함수, 생성자 함수, Object.create메서드, 클래스(!) 객체 리터럴 - var 변수명 = {프로퍼티(,로 연결)}; // 이때 {}는 코드블록이 아니므로 ; 필요 *프로퍼티 1. 키 - 키 값은 식별자 네이밍을 따르는 것이 좋다. - 아닐 경우 '', ""안에 값을 넣어준다. 빈 문자열도 가능하다.(권장되지 않음) - 키는 동적 생성 가능. ( 객체 생성, 변수 선언> 객체명[변수명]=값 ) - 키를 숫자로 지정해도 자동으로 문자열로 저장됨 - 프로퍼티 키는 중복선언 가능하며 덮어쓰기 되므로 주의 2. 접근 - 마침표 표기법: 객체명.프로퍼티키 // 명명규칙 따르지 않는 키 사용 불가 - 대괄호 표기법 객체명["프로퍼티키"] // 따옴표 필수. 없으면 undefined 3. 프로퍼티 값 갱신, 생성, 삭제 - 외부에서 값 변경 가능 - 존재하지 않는 프로퍼티 할당 시 동적 생성 - delete 객체명.프로퍼티키; // 프로퍼티 삭제. 없는 프로퍼티에 적용해도 에러 미발생 - 프로퍼티 명과 변수 이름 같은 경우 '객체명.프로퍼티키' 형식이 아니라 축약하여 프로퍼티키로 호출 가능 ex. let x=1, y=2; const obj={ x, y }; // obj 출력시 {x:1, y:2} - 프로퍼티 이름에 계산식 추가 : 대괄호 표기법 사용. 안에 i++ 등 연산자 넣어 객체['제목' + i++ ]='값'; - 위의 방식은 객체 내(ES6), 외부(ES5) 모두에서 가능 - ES6: 메서드 축약 표현으로 정의 가능. 메서드명(인자) {실행문} 형식
